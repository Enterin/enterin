Include enterin.js in your web page

To do that you only need 2 lines of code:

<script src="//ajax.googleapis.com/ajax/libs/jquery/1.10.1/jquery.min.js"></script>
<script rel="js/enterin" src="enterin/enterin.js"></script>



* enterin.js has to be included in the root of your website and it has to be called as follows:"enterin/enterin.js"

Include jQuery and enterin.js inside <head> tags:

<!DOCTYPE html>
	<html>
	<head>
		<meta charset="utf-8">
		<title>Page Title</title>

		<script src="//ajax.googleapis.com/ajax/libs/jquery/1.10.1/jquery.min.js"></script>
		<script rel="js/enterin" src="enterin/enterin.js"></script>

	</head>
	...

Get it to work!

Once you include enterin.js in your webpage, it's ready to be used!

Let's see the classes and the attributes to get enterIN work properly:

<section data-enterin="true" data-enterin-type="page">
	<figure class="enterin-slide" data-enterin-to="1"><img src="shape1.png" alt=""></figure>
	<figure class="enterin-slide" data-enterin-to="2"><img src="shape4.png" alt=""></figure>
	<figure class="enterin-slide" data-enterin-to="3"><img src="shape3.png" alt=""></figure>
	<figure class="enterin-slide" data-enterin-to="4"><img src="shape5.png" alt=""></figure>
	<figure class="enterin-slide" data-enterin-to="5"><img src="shape2.png" alt=""></figure>
</section>

Here i'll explain the needed html5 markup:

data-enterin="true"		(required)

This attribute tells enterin.js that the elements <div> or <section> contain the necessary instructions to let it work properly.

class=".enterin-slide"		(required)

All the: elements/slides/pages/images inside of <div> or <section> tags with the data-enterin="true" attribute

data-enterin-to="#"	(optional)

This attribute defines the # position of the slide. if it's clicked or tapped, it shows the # slide.

data-enterin-type="page"	(optional)

This attribute defines if enterin.js has to interprete the markup as native or page.

Configuration

Once the markup has been built, you just need to set the speed of the animation and the style of your web application.

To do so, you have to open the file enterin/config.enterin.css:

/*
*
* 	Edit the speed and the effect type of enterin.js 
*
* 	@enterin-effect-speed: 		CSS3 transition-time value
*	@enterin-effect-ease: 		CSS3 transition-ease value
*
*/

@enterin-effect-speed: 		1.5s;

@enterin-effect-ease: 		ease-in-out;

/*
*
* 	Now we can override one or more rules without editing the core.enterin.css and style.enterin.css files.
*
*	.enterin-body-css: 				Body tag css style
*	.enterin-wrapper-css: 			element[data-enterin="true"] css style
*	.enterin-slide-css: 			element[data-enterin="true"] .enterin-slide css style
*
*/

.enterin-body-css {
	
	/* css code here */

}

.enterin-wrapper-css {

	/* css code here */

}


.enterin-slide-css {

	/* css code here */
	
}

Customization

Let's go ahead with the customization section!

To customize the style of your web application you just need to edit the custom.enterin.css file.

Events

Let's see how to handle events with enterin.js

Mouse Enter:

This event triggers when you hover over a slide that has the mouseenter class.

To get the event and assign a style to it you can assign the following rule in thecustom.enterin.css file:

element[data-enterin="true"] .enterin-slide.mouseenter {
	/* your css rules */
}

enterin.js Active:

This event triggers on the current viewed slide by assigning the enterin-active rule.

To get to the event and assign a style, you can use the following rule in thecustom.enterin.css file:

element[data-enterin="true"] .enterin-slide.enterin-active {
	/* your css rules */
}

Mouse, keyboard and... fingers!

enterin.js can handle, as well, events initialized by mouse clicks, mouse wheel, key pression and finger pression with a touch screen.

Let's see them in the detail:

MOUSE:

	wheel down:					+1 to the current slide index. If it's on the last slide, it goes back to the first.
	wheel up: 					-1 to the current slide index. If it's on the first slide, it goes back to the last.
	middle button: 					Enables the "grid" mode, displaying all the slides in 2 columns.
	 						Disable it by pressing again.


KEYBOARD:

	right/down key:					+1 to the current slide index. If it's on the last slide, it goes back to the first.
	left/up key: 					-1 to the current slide index. If it's on the first slide, it goes back to the last.
	G key: 						Enables the "grid" mode, displaying all the slides in 2 columns.
	 						Disable it by pressing again.


TOUCH:

	left swipe:					+1 to the current slide index. If it's on the last slide, it goes back to the first.
	right swipe: 					-1 to the current slide index. If it's on the first slide, it goes back to the last.
	2 fingers tap: 					Enables the "grid" mode, displaying all the slides in 2 columns.
							Disable it by pressing again.

Advanced
Taking control of enterin.js

Let's see how you can recall these events by assigning them to an element or an external function.

You can choose to assign to an element the data-enterin-to="#" attribute to let the click be able to skip to the desired index.

<a href="#" data-enterin-to="10">CLICK ME TO GO TO THE 10</a> SLIDE

You can get to the same result by using javascript, as shown below:

<script>  $.EnterIN.changeSlide(10); </script>
